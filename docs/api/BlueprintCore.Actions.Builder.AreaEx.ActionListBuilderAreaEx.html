Remarks

Actions are supported using extension methods. Include the extension namespaces as needed.

When Build() is called the Kingmaker.ElementsSystem.ActionList is constructed, validated, and returned. If any errors are detected by Validator they will be logged as a warning.

Do not call Build() twice on the same builder.

If a method calls for a string to represent any type of blueprint, you can pass the blueprint's Kingmaker.Blueprints.SimpleBlueprint.AssetGuid as a string or as a name you already provided using AddGuidsByName(Dictionary<String, String>).

ActionListBuilderAreaEx Actions involving the game map, dungeons, or locations. See also ActionListBuilderKingdomEx for location related actions specifically tied to the Kingdom and Crusade system.
ActionListBuilderAVEx Actions involving audiovisual effects such as dialogs, camera, cutscenes, and sounds.
ActionListBuilderBasicEx Most game mechanics related actions not included in ActionListBuilderContextEx.
ActionListBuilderContextEx Most Kingmaker.UnitLogic.Mechanics.Actions.ContextAction types. Some Kingmaker.UnitLogic.Mechanics.Actions.ContextAction types are in more specific extensions such as ActionListBuilderAVEx or ActionListBuilderKingdomEx.
ActionListBuilderKingdomEx Actions involving the Kingdom and Crusade system.
ActionListBuilderMiscEx Actions without a better extension container such as achievements and CustomEvent.
ActionListBuilderNewEx Actions defined in BlueprintCore and not available in the base game.
ActionListBuilderStoryEx Actions related to the story such as companion stories, quests, name changes, and etudes.
ActionListBuilderUpgraderEx All UpgraderOnlyActions.
Apply a buff and make a melee attack: 
  // Provides ApplyBuff and MeleeAttack extensions
  using BlueprintCore.Actions.Builder.ContextEx; 

var actionList =
ActionListBuilder.New()
.ApplyBuff(MyAttackBuff, duration: ContextDuration.Fixed(1))
.MeleeAttack()
.build();
