Class ConditionsCheckerBuilderContextEx

Extension to ConditionsCheckerBuilder for most Kingmaker.UnitLogic.Mechanics.Conditions.ContextCondition types. Some Kingmaker.UnitLogic.Mechanics.Conditions.ContextCondition types are in more specific extensions such as KingdomEx.

Remarks

Conditions are supported using extension methods. Include the extension namespaces as needed.

When Build() is called the Kingmaker.ElementsSystem.ConditionsChecker is constructed, validated, and returned. If any errors are detected by Validator they will be logged as a warning.

Do not call Build() twice on the same builder.

If a method calls for a string to represent any type of blueprint, you can pass the blueprint's Kingmaker.Blueprints.SimpleBlueprint.AssetGuid as a string or as a name you already provided using AddGuidsByName(Dictionary<String, String>).

AreaEx Conditions involving the game map, dungeons, or locations. See also KingdomEx for location related conditions specifically tied to the Kingdom and Crusade system.
BasicEx Most game mechanics related conditions not included in ConditionsCheckerBuilderContextEx.
ConditionsCheckerBuilderContextEx Most Kingmaker.UnitLogic.Mechanics.Conditions.ContextCondition types. Some Kingmaker.UnitLogic.Mechanics.Conditions.ContextCondition types are in more specific extensions such as KingdomEx.
KingdomEx Conditions involving the Kingdom and Crusade system.
ConditionsCheckerBuilderNewEx Conditions defined in BlueprintCore and not available in the base game.
StoryEx Conditions related to the story such as companion stories, quests, name changes, and etudes.
Make a melee attack if the target is in range: 
  // Provides the MeleeAttack extension for ActionListBuilder
  using BlueprintCore.Actions.Builder.ContextEx; 
  // Provides the TargetInMeleeRange extension for ConditionsCheckerBuilder
  using BlueprintCore.Conditions.Builder.NewEx;

var actionList =
ActionListBuilder.New()
.Conditional(
ConditionsCheckerBuilder.New().TargetInMeleeRange(),
ifTrue: ActionListBuilder.New().MeleeAttack())
.build();
