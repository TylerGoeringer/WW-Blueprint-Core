## Remarks

Only one progression can be applied to a config; only call one of these functions for a config.

If the config should return the base value, no progression is needed.

Integer division is truncated: `3 / 2 = 1` (rounds down) and `-3 / 2 = -1` (rounds up).

Config progressions are deceptively named; the functions attempt to name them correctly. For easy mapping to the enum each function comment explains the progression formula and which enums are used. See also the [ContextRankConfig Calculator](https://docs.google.com/spreadsheets/d/11nQdJ7DFzS73gwR9xk3gsKbyGgtDM51yNoMv7nNYnPw/edit?usp=sharing).
